Dans le cadre de votre application de gestion et de réservation de vols, vous pouvez structurer les données de votre table utilisateur en plusieurs catégories, de la plus essentielle à la plus avancée.
Voici la liste complète des données classées par utilité pour enrichir votre base de données MySQL :
## 1. Les données d'authentification et de sécurité (Essentielles)
Ce sont les informations indispensables pour gérer les accès et la sécurité.

* id : Identifiant unique (déjà présent).
* name / firstname : Nom et prénom pour personnaliser l'interface.
* email : L'adresse de connexion et de contact (déjà présente).
* password : Le mot de passe haché (déjà présent).
* role : Pour séparer les clients des administrateurs ('user', 'admin').
* is_verified : Un booléen (0 ou 1) pour savoir si l'utilisateur a validé son email.

## 2. Les données de profil et de voyage (Pratique pour les réservations)
Stocker ces informations évite à l'utilisateur de devoir les ressaisir à chaque achat de billet d'avion.

* phone : Numéro de téléphone (utile pour les alertes SMS en cas de retard de vol).
* birth_date : La date de naissance (obligatoire pour l'émission des billets d'avion).
* passport_number : Numéro de passeport pour les vols internationaux.
* passport_expiry : Date d'expiration du passeport (pour l'avertir s'il est périmé).
* nationality : Nationalité pour les exigences de visa.

## 3. Les préférences de voyage (Pour l'expérience utilisateur)
Ces données permettent de pré-remplir les filtres de recherche ou de personnaliser les propositions de vols.

* preferred_class : Classe favorite ('economy', 'business', 'first').
* preferred_seat : Préférence de siège ('window' pour hublot, 'aisle' pour couloir).
* dietary_restrictions : Repas à bord (végétarien, halal, sans gluten).
* frequent_flyer_number : Numéro de programme de fidélité ou de alliance aérienne.

## 4. Le suivi et l'historique (Métadonnées)
Ces champs servent à l'administration, aux statistiques ou à la sécurité de l'application.

* created_at : Date d'inscription (déjà présente).
* updated_at : Date de la dernière modification du profil.
* last_login : Date et heure de la dernière connexion.

------------------------------
## ⚠️ Règle d'or de sécurité (Le Payload du JWT)
Même si vous stockez toutes ces données dans votre table MySQL, ne les mettez pas toutes dans votre jeton JWT.
Le JWT transite à chaque requête HTTP et peut être décodé par n'importe qui. Mettez-y uniquement le strict minimum :

* Oui dans le JWT : id, email, role.
* Non dans le JWT : password (jamais !), passport_number, phone.

Souhaitez-vous obtenir la requête SQL (ALTER TABLE) pour ajouter certaines de ces colonnes (comme le téléphone ou la date de naissance) à votre table actuelle ?
